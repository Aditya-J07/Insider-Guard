## Overview

A lightweight, portable Insider Threat SOC Simulator designed for demonstrations. The system consists of a Python-based endpoint agent that monitors a sensitive directory for suspicious activity (tampering, deletion, rogue encryption, exfiltration tools), sending alerts to a FastAPI backend. The backend persists these alerts in an in-memory mock collection. A React/Vite frontend dashboard visualizes the alerts and provides interactive SOC capabilities (like isolating hosts or restoring access).

## Components

### Endpoint Agent
- **Purpose:** Monitors a specific local directory (`SENSITIVE_DATA`) for changes, detecting suspicious file activity (creation, modification, deletion) and decoy trigger events.
- **Location:** `agent.py`

### Backend Service
- **Purpose:** Exposes REST API endpoints (`/receive-log`, `/view-logs`, `/heartbeat`) to collect telemetry from the agent and serve it to the frontend dashboard.
- **Location:** `backend/` (`main.py`, `database.py`)
- **Dependencies:** `fastapi`, `uvicorn`, `pydantic`

### Frontend Dashboard
- **Purpose:** Provides a UI for security analysts to monitor incoming alerts, view user profiles, and take simulated automated actions.
- **Location:** `frontend/`
- **Dependencies:** `react`, `react-router-dom`, `tailwindcss`, `axios`, `lucide-react`

- **Purpose:** Windows batch scripts to streamline running hackathon demonstrations (e.g., locking access, resetting demo state, simulating ransomware).

## Data Flow

1. **Detection:** The `agent.py` script detects a file event in `SENSITIVE_DATA` using `watchdog`.
2. **Analysis:** The agent inspects the file name, extension, and magic bytes for malicious indicators (e.g., encryption tools, honeytokens).
3. **Transmission:** The agent sends an HTTP POST request with the alert payload to the backend at `http://127.0.0.1:8000/receive-log`.
4. **Storage:** The backend receives the payload and stores it in the in-memory `MockCollection` (`database.py`).
5. **Visualization:** The React frontend periodically fetches the logs from `http://127.0.0.1:8000/view-logs` using Axios and renders them on the dashboard.

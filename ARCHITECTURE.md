## Overview
The Insider Threat SOC Simulator is a client-server application designed to simulate, detect, and visualize insider threats in a Security Operations Center (SOC) environment. It features a React-based frontend providing the dashboard UI and a Python FastAPI backend handling log ingestion, behavioral analysis, risk scoring, and automated responses (e.g., locking users).

## Components

### Frontend Dashboard
- **Purpose:** Displays real-time logs, risk scores, and alerts.
- **Location:** `frontend/src/`
- **Dependencies:** React, TailwindCSS, Axios, Vite.
- **Dependents:** End users (SOC Analysts).

### Backend Risk Engine
- **Purpose:** Ingests simulation logs, performs rule-based and behavioral checks (velocity, geo-intel, honeytokens), and maintains state.
- **Location:** `backend/`
- **Dependencies:** FastAPI, SQLAlchemy, Requests.
- **Dependents:** Frontend Dashboard, Demo Automation Scripts.

### Demo Automations
- **Purpose:** Batch scripts simulating different insider threat scenarios (exfiltration, mass deletion, honeytoken triggers).
- **Location:** `Demo_Automations/`
- **Dependencies:** Windows Batch, cURL.
- **Dependents:** Backend API.

## Data Flow
1. **Simulation:** Demo scripts send mock logs to backend `/receive-log`.
2. **Ingestion & Analysis:** Backend processes the log, evaluates risk (velocity, geographic anomalies, honeytoken access).
3. **Storage:** Log and assigned risk score are saved to the local SQLite database via SQLAlchemy.
4. **Visualization:** Frontend polls or fetches `/view-logs` to render the dashboard.
5. **Enforcement:** Users reaching a threshold or touching a honeytoken are added to a blocked list, preventing further active processing.

## Conventions
- **Naming:** CamelCase for React components, snake_case for Python logic.
- **Structure:** Distinct separation of frontend and backend. Backend logic is centralized in `main.py` with DB connections in `database.py`.
- **Testing:** E2E validation primarily driven by `.bat` scripts in `Demo_Automations`.
